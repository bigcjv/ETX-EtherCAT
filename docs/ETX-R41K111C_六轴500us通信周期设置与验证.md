# ETX-R41K111C 六轴 500 us 通信周期设置与验证

## 1. 目标与不变量

本流程用于 ETX-R41K111C 在 500 us 主站周期下控制 6 台 Panasonic MINAS A6B
EtherCAT 驱动器。以下配置必须同时一致：

| 配置项 | 500 us 对应值 |
|---|---:|
| Windows `C:\TPM\ECPW\Config\Setting.json` | `"CycleTime": 500` |
| ETX `/usr/lib/ECPL/Config/Setting.json` | `"CycleTime": 500` |
| 六台驱动器 ENI `CycleTime0/1` | `500000 ns` |
| 六台驱动器 DC 周期寄存器写入 | `20A1070000000000` |
| DC 模式 | `DC SYNC0`、`x1` |

禁止只修改一个 `Setting.json`、只修改第一台驱动器，或手改 ENI 周期字节后直接运行。

## 2. 安全准备

1. 确认六轴完全停止并 Servo OFF。
2. 确认急停、正负限位和安全回路有效。
3. 停止 Scenario 1 自研程序；驱动器带电时不要重启 ETX。
4. 关闭 Windows 上的 ECATNavi 和 ECATScan。

## 3. 同步 Windows 与 ETX 主站周期

在仓库根目录运行：

```powershell
.\scripts\deploy_etx_6axis_csp.ps1 `
  -Password "<ETX_PASSWORD>" `
  -CycleTimeUs 500 `
  -PrepareEniExport `
  -ConfirmAxesStoppedAndServoOff
```

脚本会：

1. 备份并设置 Windows `Setting.json` 为 500 us；
2. 停止 Scenario 2 服务并确认自研程序未运行；
3. 备份并设置 ETX `Setting.json` 为 500 us；
4. 启用 Scenario 2，等待 TCP 5886 开始监听。

## 4. 在 ECATNavi 设置六轴 DC

1. 重新启动 ECATNavi，连接 `192.168.0.10`。
2. 扫描并确认共有 6 台 `MADLT05BF`。
3. 打开 Drive 1 的 `Device Settings -> DC`。
4. `Operation Mode` 选择 `DC SYNC0`。
5. 确认只读的 `Master Cycle Time` 显示 `500 us`。
6. `Sync0 Cycle Time` 选择 `x1`，结果应为 `500 us`。
7. 点击 `Apply to All`，再点击 `Save`。
8. **逐个打开 Drive 2 到 Drive 6 的 DC 页面**，确认每一台的 Master Cycle 和
   Sync0 都是 `500 us`。不能只看 Drive 1。
9. 回到工程界面重新生成并导出 `ENI.xml`。
10. 用导出文件替换仓库 `6axis_eni\ENI.xml`。

如果 Drive 1 是 500 us、Drive 2–6 仍是 1000 us，说明 `Apply to All` 没有真正
保存成功。不要部署；重新执行第 3–9 步。

## 5. ENI 验收

部署脚本会在连接 ETX 前自动拒绝不合格 ENI。也可先手动查看：

```powershell
[xml]$eni = Get-Content .\6axis_eni\ENI.xml -Raw
$eni.SelectNodes('//Slave/DC') | ForEach-Object {
  [pscustomobject]@{
    CycleTime0 = $_.CycleTime0
    CycleTime1 = $_.CycleTime1
    ReferenceClock = $_.ReferenceClock
  }
}
```

验收条件：

- 恰好 6 台 `MADLT05BF`；
- 恰好 6 个 `<DC>` 节点；
- 六台 `CycleTime0/1` 全部为 `500000`；
- 只有 Drive 1 的 `ReferenceClock=true`；
- 六台 DC activation 全部为 `0003`；
- 六台周期寄存器写入全部为 `20A1070000000000`。

## 6. 切换 Scenario 1 并执行不运动检查

ENI 验收通过后运行：

```powershell
.\scripts\deploy_etx_6axis_csp.ps1 `
  -Password "<ETX_PASSWORD>" `
  -CycleTimeUs 500 `
  -ConfigureCycleTime `
  -ActivateScenario1 `
  -Build `
  -RunDryCheck
```

该流程会先编译，再禁用 `etx.service`，备份并确认 ETX 周期为 500 us，安装 ENI，
最后运行不带 `--enable-motion` 的程序。必须确认：

- 连接到 6 个 CiA402 从站；
- 六轴均支持 CSP 且没有驱动报警；
- EtherCAT 能进入 OP；
- 没有 DC、WKC、看门狗或 `Err81.0` 错误。

## 7. 500 us 运动运行方法

只有 dry check 通过，并再次确认急停、限位、安全空间、六轴方向、user unit 换算、
速度和加速度后，才执行一次保守往返：

```powershell
.\scripts\deploy_etx_6axis_csp.ps1 `
  -Password "<ETX_PASSWORD>" `
  -CycleTimeUs 500 `
  -ActivateScenario1 `
  -Build `
  -RunDryCheck `
  -RunMotion `
  -ConfirmMotionSafety
```

程序仍要求显式 `--enable-motion` 才会运动，默认只执行一次六轴 Group 0 同步往返。
不得假设 `distance`、`feed`、`accel`、`decel` 的 user unit 是脉冲、毫米或角度。

## 8. 停止与恢复

- 中断或异常时，程序会 Group smooth stop、Group disable、Servo OFF、
  `ECPWDisconnect()` 和 `ECPWClose()`。
- Scenario 1 运行期间保持 `etx.service` disabled/inactive。
- Windows 和 ETX 的 `Setting.json` 备份使用 `.bak.<时间戳>` 后缀；恢复时必须同时
  恢复匹配周期的 ENI，不能只恢复其中一个文件。
