# ETX-R41K111C 控制 6 个松下 A6 EtherCAT 电机配置与调试步骤

适用产品：

- 厂商：TPM / Taiwan Pulse Motion / 台湾脉动股份有限公司
- 控制器型号：ETX-R41K111C
- 机身标识：ETX-R41K111C-4P6
- 料号：P/N 90-BR40112-130
- 类型：ARM 架构独立式 EtherCAT 主站 / 多轴运动控制器
- 从站：6 个松下 Panasonic MINAS A6 EtherCAT 伺服驱动器

参考手册：

- `ETX-R41K111_Series_UserManual_1.0.pdf`
- `ETX-R41K111_Series_ECATNaviManual_1.0.pdf`

> 目标：先通过 ECATNavi/ECATScan 把 6 个松下 A6 驱动器扫描出来，生成 EtherCAT 网络配置文件 `ENI.xml`，然后逐轴完成 Servo ON、报警清除、点动、回原点和简单定位，最后再做 6 轴编组或应用程序控制。

## 1. 准备清单

### 1.1 必备文件

1. ETX-R41K111C 软件包，通常包含：
   - `EtxTool`
   - `ECATScan`
   - `ECATNavi`
   - ECPW / ECPL 运行库
2. 松下 A6 EtherCAT 驱动器的 ESI 文件，通常是 `.xml` 文件。
   - 文件由松下提供，名称可能类似 `Panasonic...A6...EtherCAT.xml`。
   - 如果没有 ESI，ECATScan 扫描时会出现 unknown device 或无法正确生成轴对象。
3. 松下 A6 驱动器手册，用来确认：
   - 电子齿轮比 / 每转脉冲数
   - 原点方式 `0x6098`
   - 限位、急停、报警代码
   - 支持的 CiA402 模式：CSP / PP / Homing

### 1.2 硬件连接

1. ETX-R41K111C 接 DC 24V 电源。
2. PC 网口连接 ETX-R41K111C 的 Ethernet 口，用于 Windows 场景下配置和控制。
3. ETX-R41K111C 的 EtherCAT 口连接第 1 台 A6 驱动器的 EtherCAT IN。
4. 第 1 台 A6 的 EtherCAT OUT 连接第 2 台 IN，依次串接到第 6 台。
5. 每台 A6 接好：
   - 主电源 / 控制电源
   - 电机动力线
   - 编码器线
   - 必要的 Servo ON、安全回路、急停、限位信号

建议第一次调试时不要带机械负载，或确保电机脱开负载、急停有效、限位有效。

## 2. 确认使用场景

ETX-R41K111C 支持两种使用方式：

| 场景 | 说明 | 常见用途 |
|---|---|---|
| Scenario 1 | 控制程序直接运行在控制器 Linux 系统上 | 独立运行 |
| Scenario 2 | Windows PC 通过 Ethernet 连接 ETX-R41K111C | 初次调试、Windows 上位机开发 |

你现在说“下载并安装了软件”，大概率先按 Scenario 2 操作，也就是 Windows PC + ECATNavi 调试。

## 3. 配置 ETX-R41K111C IP

手册默认参数：

| 项目 | 默认值 |
|---|---|
| IP Address | `192.168.0.10` |
| Subnet Mask | `255.255.255.0` |
| Gateway | `192.168.0.0` |

操作步骤：

1. PC 网口直连 ETX-R41K111C 的 Ethernet 口。
2. 把 PC 网卡设置到同一网段，例如：
   - IP：`192.168.0.100`
   - 子网掩码：`255.255.255.0`
   - 网关可空或填 `192.168.0.1`
3. 打开 `EtxTool`。
4. 进入 `IP Setting` 页。
5. 选择连接 ETX-R41K111C 的 PC 网卡，点击 `OPEN`。
6. 如果需要修改控制器 IP，填写新的 IP / Subnet Mask / Gateway，然后点击 `Set`。
7. 确保 ETX-R41K111C 和 PC 在同一网段，并且 IP 不冲突。

如果不确定是否连接成功，可以先在 Windows 命令行执行：

```powershell
ping 192.168.0.10
```

## 4. 确认控制器场景为 Scenario 2

如果你要用 Windows 的 ECATNavi 调试：

1. 打开 `EtxTool`。
2. 输入 ETX-R41K111C 的 IP。
3. 点击 `Current Scenario` 查看当前场景。
4. 如果不是 Windows-based / Scenario 2，点击 `Use S2 Windows-based`。
5. 切换完成后重新连接 ECATNavi。

如果后面要让控制器脱离 PC 独立运行，再切回 Scenario 1 并把应用程序部署到控制器 Linux 环境。

## 5. 放置松下 A6 的 ESI 文件

ECATScan 生成 ENI 之前必须能读取从站 ESI。

Windows Scenario 2 默认 ESI 路径：

```text
C:\TPM\ECPW\ESI
```

Linux Scenario 1 默认 ESI 路径：

```text
/usr/lib/ECPL/ESI
```

操作步骤：

1. 找到松下 A6 EtherCAT 驱动器的 ESI `.xml` 文件。
2. 复制到 `C:\TPM\ECPW\ESI`。
3. 打开 `ECATNavi`。
4. 点击菜单栏 `Scan` 打开 ECATScan。
5. 点击 `Reload ESI`。
6. 在左侧 `Available ESI Devices` 中确认能看到松下 A6 相关设备。

如果扫描后出现未知设备，通常是 ESI 文件不匹配。需要确认 ESI 的 Vendor ID、Product Code、Revision 是否覆盖你手上的 A6 EtherCAT 型号。

## 6. 扫描 6 个 A6 从站并生成 ENI

### 6.1 在线扫描

1. 确认 6 台 A6 驱动器都上电。
2. 确认 EtherCAT 网线顺序：
   - ETX EtherCAT port -> A6-1 IN
   - A6-1 OUT -> A6-2 IN
   - ...
   - A6-5 OUT -> A6-6 IN
3. 在 ECATNavi 点击 `Scan`。
4. 在 ECATScan 窗口点击 `Reload ESI`。
5. 点击 `Scan`。
6. 右侧 `Selected Devices` 应看到 6 个松下 A6 从站。
7. 记录显示顺序。EtherCAT 拓扑顺序通常就是轴号顺序：
   - 第 1 个 A6 -> Axis 0 或 Axis 1，按软件显示为准
   - 第 2 个 A6 -> 下一轴
   - ...
   - 第 6 个 A6 -> 最后一轴

### 6.2 检查设备设置

1. 双击右侧某个 A6 设备，打开 `Device Setting`。
2. 初次调试建议先使用 ESI 默认设置，不要急着改 PDO。
3. 如果必须改 PDO：
   - 确保包含 CiA402 常用对象，例如 Controlword `0x6040`、Statusword `0x6041`、Modes of operation `0x6060`、Position actual value `0x6064`、Target position `0x607A` 等。
   - 6 台相同 A6 可以使用 `Apply to All` 应用到同型号设备。
4. 点击 `Save` 保存设置。

### 6.3 生成并连接 ENI

有两种常用方式：

方式 A：快速连接

1. 在 ECATScan 菜单栏点击 `Quick Connect`。
2. 软件会把当前设备设置导出为 `ENI.xml`，并立即连接。
3. 注意：这会覆盖原来的同名 `ENI.xml`。

方式 B：先导出再连接

1. ECATScan 菜单选择 `File -> Export ENI`。
2. 保存到默认 ENI 目录。
3. Windows Scenario 2 默认 ENI 路径：

```text
C:\TPM\ECPW\ENI
```

4. 回到 ECATNavi，点击 `Connect`。
5. 默认会查找 `ENI.xml`。
6. 如果要选择其他 ENI，点击 `Connect` 旁边的小三角，选择 `Connect with other ENI`。

连接成功后，左侧树形列表会显示 Slave、Axis、AxisGroups 等对象。

## 7. 单轴首次调试流程

不要一开始就让 6 个轴一起动。建议按下面顺序一轴一轴确认。

### 7.1 打开轴控制窗口

1. 在 ECATNavi 左侧树形列表点击第 1 个 Axis。
2. 右侧或弹窗会显示该轴的控制窗口和属性面板。
3. 查看 Monitor 面板：
   - `ActPos`
   - `Vel`
   - `Motion Status`
   - `SVON`
   - `InPos`
   - `PEL/NEL`
   - `Driver Error Code`

如果 `Driver Error Code` 有值，需要查松下 A6 手册处理报警。

### 7.2 清报警

1. 点击 `Clear Alarm`。
2. 确认驱动器报警消失。
3. 如果报警清不掉，先处理驱动器侧硬件报警，例如急停、安全输入、编码器、电源、限位。

### 7.3 Servo ON

1. 确认电机固定安全，急停可用。
2. 点击 `Servo`。
3. 查看 `SVON` 标志是否变为 ON。
4. 如果 Servo ON 失败：
   - 检查松下 A6 使能条件
   - 检查安全输入
   - 检查控制电源和主电源
   - 检查 EtherCAT 状态是否进入 OP

### 7.4 Reset Pos

如果只是空载试转，可先点击 `Reset Pos`，把当前位置置零。

注意：`Reset Pos` 不是完整回原点，只是把当前位置设为零。正式设备仍应配置 Home。

### 7.5 Jog 点动

1. 进入 `Jog` 页。
2. 设置低速参数，例如：
   - Speed：先用很小的用户单位速度
   - Acc：先用较小加速度
3. 按住 `Positive` 或 `Negative`。
4. 松开按钮，轴应停止。
5. 确认方向：
   - 正方向是否符合机械定义
   - 编码器位置是否随运动方向变化
   - 限位方向是否正确

如果方向相反，优先按机械/驱动器参数统一方向，不要在不同层随意反号。

### 7.6 Move 定位测试

1. 进入 `Move` 页。
2. 优先选择 `CSP`，如果驱动器或 ESI 不支持，再选 `PP`。
3. 选择 `REL` 相对运动。
4. 输入一个很小的位移，例如 `100` 或更小，具体取决于你的用户单位。
5. 设置低速和低加速度。
6. 点击 `Move To`。
7. 观察：
   - `ActPos` 是否接近目标
   - `InPos` 是否到位
   - 是否有 following error
   - 电机是否异常振动或方向错误

确认第 1 轴正常后，再按同样步骤调第 2 到第 6 轴。

## 8. 回原点 Home 配置

Home 页对应 CiA402 Homing Mode，关键参数来自松下 A6 手册。

常见参数：

| 参数 | CiA402 对象 | 说明 |
|---|---|---|
| Method | `0x6098` | 原点方式 |
| Spd 1 | `0x6099:01` | 找开关速度 |
| Spd 2 | `0x6099:02` | 找零点速度 |
| Accel | `0x609A` | 原点加速度 |
| Offset | `0x607C` | 原点偏移 |

操作步骤：

1. 查松下 A6 手册，确认你的机械原点方式。
2. 在 ECATNavi 的 Axis -> `Home` 页填写 Method、速度、加速度、Offset。
3. 点击 `Start Home`。
4. 观察 Statusword：
   - bit 10：Target reached
   - bit 12：Homing attained
   - bit 13：Homing error
5. 如果 Homing error，检查：
   - 原点传感器信号
   - 正负限位
   - 原点方向
   - 松下 A6 对该 Method 的定义

6 个轴每个都要单独验证原点，不能只验证一个轴后直接复制到所有轴。

## 9. 6 轴编组控制

单轴都确认无误后，再使用 AxisGroups。

操作步骤：

1. 在左侧树形列表点击 `AxisGroups`。
2. 在 `Available Axes` 中选择 6 个轴。
3. 加入到目标 Group 的 `Joined Axes`。
4. 在 Group 面板中可执行：
   - `GroupServoON / OFF`
   - `GroupClearAlarm`
   - `GroupStop`
   - `GroupHold / Resume`
   - `GroupSetFeedrate`
   - 直线插补、圆弧插补等群组运动

建议顺序：

1. 先 2 轴 group 做低速直线测试。
2. 再扩到 3 轴。
3. 最后扩到 6 轴。

任何一轴报警或限位异常，都先停止 group，不要继续插补测试。

## 10. 正式应用前必须确认的参数

### 10.1 用户单位

ECATNavi 里的位置、速度、加速度显示为 user unit。正式控制前要统一：

- 1 user unit 等于多少脉冲、多少 mm、多少度？
- A6 电子齿轮比是多少？
- 机械丝杆导程 / 减速比是多少？
- 旋转轴是否需要 ring counter？

建议建立轴参数表：

| 轴号 | 机械名称 | EtherCAT 顺序 | 方向 | 每转单位 | 限位 | 原点方式 |
|---|---|---:|---|---:|---|---|
| Axis 1 | X | 1 | 正向待确认 | 待填 | 待填 | 待填 |
| Axis 2 | Y | 2 | 正向待确认 | 待填 | 待填 | 待填 |
| Axis 3 | Z | 3 | 正向待确认 | 待填 | 待填 | 待填 |
| Axis 4 | A | 4 | 正向待确认 | 待填 | 待填 | 待填 |
| Axis 5 | B | 5 | 正向待确认 | 待填 | 待填 | 待填 |
| Axis 6 | C | 6 | 正向待确认 | 待填 | 待填 | 待填 |

### 10.2 运动模式

ETX-R41K111C 支持 CiA402：

- PP：Profile Position Mode
- CSP：Cyclic Synchronous Position Mode
- Homing：回原点模式

一般建议：

- 调试单点定位：PP 或 CSP 都可。
- 多轴插补：优先 CSP。
- 回原点：使用 Homing，并按松下 A6 手册设置 `0x6098` 等参数。

### 10.3 ENI 文件管理

每次修改 ESI/PDO/设备顺序后都要重新生成 ENI。

建议保存多个 ENI：

```text
ENI_6Axis_A6_initial.xml
ENI_6Axis_A6_pdo_default.xml
ENI_6Axis_A6_final.xml
```

调试时可以通过 `Connect -> 小三角 -> Connect with other ENI` 指定文件。

### 10.4 SDO 参数保存

ECATNavi 的 `SDO List` 可读写从站 SDO。

如果修改了驱动器参数并希望断电保持：

1. 打开对应 Slave 的 `SDO List`。
2. 修改需要的 SDO。
3. 点击 `Write To EEPROM`。

注意：写 EEPROM 前必须确认参数正确，尤其是方向、限位、原点、电子齿轮、PDO 相关参数。

## 11. 推荐的完整调试顺序

按这个顺序做，最稳：

1. 检查 24V、电机电源、安全回路、急停。
2. PC 设置到 `192.168.0.x` 网段。
3. 用 `EtxTool` 确认 ETX-R41K111C IP。
4. 用 `EtxTool` 确认当前为 Scenario 2。
5. 把松下 A6 ESI 文件放入 `C:\TPM\ECPW\ESI`。
6. 打开 ECATNavi，输入控制器 IP 并连接软件。
7. 打开 `Scan`，点击 `Reload ESI`。
8. 点击 `Scan`，确认出现 6 个 A6 从站。
9. 使用默认 PDO，点击 `Quick Connect` 生成 `ENI.xml` 并连接。
10. 逐轴打开 Axis 面板。
11. 每轴先 `Clear Alarm`。
12. 每轴 `Servo ON`。
13. 每轴 `Reset Pos`。
14. 每轴低速 `Jog`，确认方向。
15. 每轴低速 `Move REL`，确认定位。
16. 每轴配置并测试 `Home`。
17. 建立 2 轴 Group，低速插补测试。
18. 扩展到 3 轴、6 轴 Group。
19. 保存最终 ENI 和轴参数表。
20. 再开始写正式上位机或控制器端应用程序。

## 12. 常见问题排查

### 12.1 Scan 不到从站

检查：

- EtherCAT 网线是否接在 ETX 的 EtherCAT 口，不是普通 Ethernet 口。
- A6 驱动器是否上电。
- IN/OUT 是否接反。
- 是否有某个中间从站断电导致后面的轴都扫不到。
- ESI 是否放在 `C:\TPM\ECPW\ESI` 并已 `Reload ESI`。

### 12.2 扫描到 unknown device

原因通常是 ESI 不匹配。

处理：

- 找对应 A6 EtherCAT 型号和固件版本的 ESI。
- 确认 ESI 里的 Vendor ID、Product Code、Revision 覆盖实物。
- 重新放入 ESI 目录并点击 `Reload ESI`。

### 12.3 Connect 失败

检查：

- 是否已经生成 `ENI.xml`。
- `ENI.xml` 是否在 `C:\TPM\ECPW\ENI`。
- 当前物理从站数量和顺序是否与 ENI 一致。
- 之前生成的 ENI 是否被旧配置覆盖。
- 是否需要用 `Connect with other ENI` 选择正确文件。

### 12.4 Servo ON 失败

检查：

- 松下 A6 是否有报警。
- 安全输入是否满足。
- 主电源是否接入。
- EtherCAT 状态是否进入 OP。
- Statusword `0x6041` 和 Error Code `0x603F`。

### 12.5 Jog 方向反了

处理原则：

- 先统一机械定义。
- 再统一驱动器方向参数。
- 最后才考虑软件层方向系数。

不要让单轴调试、回原点、插补程序各自使用不同方向规则。

### 12.6 Home 失败

检查：

- `0x6098` Homing Method 是否适合松下 A6 和实际接线。
- 原点传感器是否进了正确输入。
- 正负限位是否有效且方向正确。
- Homing Speed 和 Accel 是否过大。
- Statusword bit 13 是否为 Homing error。

## 13. 最小验收标准

在进入正式程序开发前，至少满足：

- ECATScan 能稳定扫描出 6 个 A6。
- ECATNavi 能用固定 ENI 稳定连接。
- 6 个轴都能 Servo ON / Servo OFF。
- 6 个轴都能 Clear Alarm。
- 6 个轴都能低速 Jog。
- 6 个轴方向和机械定义一致。
- 6 个轴都能完成小距离相对定位。
- 6 个轴都能正确回原点。
- 急停、限位、报警处理有效。
- 最终 ENI、ESI、轴参数表已归档。
